# Moorhuhn-Style Shooter on `/play` — Design Spec

**Date:** 2026-07-05
**Status:** Approved design, pending implementation plan

## Goal

A small nostalgic Moorhuhn-style shooter on a dedicated `/play` page of danielariel.com. Classic cartoon chickens, reimagined in a twilight-moor scene that blends into the site's dark, emerald-accented aesthetic. Small scope, zero new runtime dependencies.

## Architecture

Four focused units under `components/game/`:

| File | Responsibility | Depends on |
|---|---|---|
| `engine.ts` | Pure TS game logic: world state, `update(state, dt, events) → state`, spawning, hit testing, ammo/reload state machine, scoring, round timer. No DOM, no React. | nothing |
| `sprites.ts` | Art authored as inline SVG strings; rasterized once at load into `ImageBitmap` sprites (per depth-layer size and wing frame). | DOM (load time only) |
| `audio.ts` | Web Audio synthesized SFX: noise-burst shotgun blast, two-step click-clack reload, pitch-bent squawk, end-of-round jingle. Empty-chamber click when firing with 0 shells. Mute state read/written to `localStorage`. | Web Audio API |
| `MoorhuhnGame.tsx` | Only client React component: owns `<canvas>`, `requestAnimationFrame` loop, input handling, HTML overlays (start / pause / end screens, HUD buttons). | engine, sprites, audio |

React renders the shell and overlays; the game loop renders the canvas directly (no per-frame React state updates).

## Gameplay rules

- **Round:** 90 seconds. Ends when the timer expires.
- **Ammo:** 8 shells. Reload via **R** key, clicking/tapping the shell HUD, or the on-screen reload button. Reload takes 0.6 s, during which shooting is disabled; on completion all 8 shells are refilled. Firing with 0 shells plays an empty click and shows a brief "Reload!" hint.
- **Chickens:** spawn from either screen edge at a random height in one of three depth layers:
  - far (small, 15 pts), mid (10 pts), near (large, 5 pts)
  - Layer determines sprite scale, speed range, and vertical band.
- **Spawning:** average spawn interval ramps from ~1.5 s at round start to ~0.8 s at round end; hard cap of 6 chickens on screen. Chickens that exit the far edge despawn with no penalty.
- **Hits:** a hit drops the chicken (rotate + fall + feather puff) and shows a floating "+N" score label. Misses have no penalty beyond the spent shell.
- **Scoring persistence:** personal best stored under `localStorage` key `moorhuhn.best`. End screen shows round score, personal best, a "New personal best!" state when beaten, and a replay button.

## Look & feel

- **Scene:** sky gradient from site background `#0a0a0a` down to a deep emerald-tinted horizon (derived from accent `#34d399`, heavily darkened); moon, sparse stars, 2–3 layers of hill silhouettes drifting slowly for parallax; the site's film-grain texture overlaid.
- **Chickens:** warm-colored (browns, red comb) goofy cartoon style so they pop against the dark scene; 2-frame wing-flap animation; authored as SVG, drawn as sprites.
- **Feedback:** crosshair cursor inside the canvas, muzzle flash and subtle screen shake per shot.
- **HUD:** Geist Mono, emerald accents, matching existing site tokens. Shells bottom-left, timer top-center, score top-right, mute toggle top-left, reload button bottom-right (always visible; primary reload affordance on touch).
- **HUD rendering split:** per-frame readouts (shells, timer, score) are drawn on the canvas each frame using the Geist Mono font — no React state updates during gameplay. Interactive controls (mute toggle, reload button, start/replay/resume) are HTML buttons overlaid on the canvas; they change only on user interaction.

## Input & responsiveness

- **Desktop:** mouse aims the crosshair, left click shoots, **R** reloads, **Esc** pauses.
- **Touch:** tap shoots at the tap point; persistent on-screen reload button (min 44 px target).
- **Canvas:** fixed virtual resolution 1280×720; scaled to fit its container up to ~1100 px wide, centered, DPR-aware for sharpness. All game coordinates are in virtual units, so gameplay is identical at every physical size.

## Edge cases & accessibility

- `prefers-reduced-motion`: disable screen shake and parallax drift; chicken movement remains (it is the game).
- `visibilitychange` / window blur mid-round: auto-pause with a resume overlay; the round timer does not advance while paused.
- Start, replay, pause-resume, and mute controls are real focusable HTML buttons. End-of-round score is announced via an `aria-live` region. Aiming is inherently pointer-based; the start screen states the controls honestly rather than claiming keyboard playability.
- SSR: the page shell is server-rendered; the game initializes inside `useEffect`. No canvas access during render, no hydration mismatch.
- If `localStorage` is unavailable (private mode), the game runs normally without persistence.

## Testing

Engine-level unit tests with the existing Vitest setup (`npm test`):

- Reload state machine: cannot shoot while reloading; refills to exactly 8; empty-chamber shot does not decrement below 0.
- Scoring: correct points per depth layer; score accumulates.
- Timer: round ends at 0; no state updates after end.
- Spawning: interval stays within ramp bounds; on-screen cap of 6 respected.
- Hit testing: point-in-chicken geometry, including scaled far-layer hitboxes.
- Personal best: storage access wrapped in a small helper so tests can mock it; new-best detection logic covered.

Rendering and audio are excluded from unit tests; verified manually by playing.

## Out of scope

- Global/server leaderboard (may be added later; localStorage design does not block it)
- Keyboard-only aiming / full gamepad support
- Faithful daylight Moorhuhn art or licensed assets
- Sound enabled by default
- Multiple rounds, levels, or power-ups
