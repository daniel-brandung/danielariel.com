# Moorhuhn 3D — "The Third Dimension" — Design Spec

**Date:** 2026-07-06
**Status:** Approved

## Goal

Add a 3D Moorhuhn-style shooter as a sibling to the existing 2D game, and turn
`/play` into a hub that hosts both. Procedural low-poly visuals, look-around
panorama camera, desktop + touch input, and extra "juice" (particles, combo
multiplier, golden chicken, day/night sky). No online leaderboard in this
version.

## Routes & pages

- `/play` — new **hub page** (server component): title + two game cards
  ("Moorhuhn Classic" and "Moorhuhn 3D"), each with a short tagline and its
  personal best rendered client-side.
- `/play/classic` — the existing 2D game page moves here as-is (content and
  metadata preserved).
- `/play/3d` — the new game.
- The footer link keeps pointing at `/play`; bookmarked visitors land on the
  hub, one click from the classic game.

## Architecture

New directory `components/game3d/`. One new dependency: `three`
(+ `@types/three` dev dependency). The game component is loaded client-only so
Three.js never enters any other route's bundle.

| File | Role |
|---|---|
| `components/game3d/engine3d.ts` | Pure TS, **no Three.js import**. Immutable `GameState3D`, `update(state, dt, rand)`, `shoot(state, ray)`, event emission mirroring the 2D engine (`shot`, `hit`, `empty`, `reloadStart`, `reloadEnd`, `roundEnd`, plus `combo` and `goldenSpawn`). Chickens live in plain 3D world coordinates; hit testing is ray-vs-sphere math with hand-rolled vector helpers so the engine is vitest-testable without WebGL. |
| `components/game3d/Moorhuhn3D.tsx` | Thin client component: owns the Three.js renderer, rAF loop, camera, input, DOM HUD overlay (styled like the 2D HUD), pause-on-blur, aria-live region, mute toggle — the same responsibilities `MoorhuhnGame.tsx` has for the 2D game. |
| `components/game3d/world.ts` | Procedural scene builders: low-poly terrain disc, pine trees, a windmill with rotating blades, gradient sky dome, fog. Also exports pure day/night math (`skyColorsAt(progress)`) so the color ramp is unit-testable without Three. |
| `components/game3d/chicken.ts` | Low-poly chicken mesh from primitives (body, wings, beak, comb) with a wing-flap animation. Golden variant reuses the mesh with an emissive gold material. |

Reused/extended from the 2D game:

- `components/game/audio.ts` — extended with 2–3 new synthesized sounds
  (combo ding, golden fanfare).
- `components/game/storage.ts` — generalized to take a storage key so classic
  and 3D keep separate personal bests. The existing key remains unchanged so
  no player loses their classic best.

## Gameplay

Same round skeleton as classic: **90-second round, 8 shells, 0.6 s reload,
phases ready → playing → paused → ended**, personal best persisted locally.

- **Depth scoring:** chickens spawn in three distance bands (near / mid / far
  from the camera). Far = 15, mid = 10, near = 5 — identical values to the 2D
  layers, but depth is now real.
- **Combo multiplier (new):** consecutive hits raise the multiplier
  ×1 → ×2 → ×3 → ×4 (cap). A missed shot or an `empty` click resets it to ×1.
  The HUD shows the current multiplier. Points awarded = band points ×
  multiplier.
- **Golden chicken (new):** rare spawn (~1 per 20 spawns, guaranteed at least
  once mid-round), flies faster on a weaving path, worth 50 × multiplier, with
  a distinct fanfare and gold particle burst.
- **Juice:** feather-explosion particle bursts at the hit point (instanced
  quads; the engine emits the event, the renderer owns particle lifetimes),
  floating score popups that face the camera. Camera shake is deliberately
  skipped — it fights with aiming.

## 3D scene & visuals

Flat-shaded procedural low-poly: terrain disc with gentle hills, scattered
pine trees, a windmill, gradient sky dome, distance fog. **Day/night cycle:**
over the 90-second round the sky transitions dusk → night — sky, fog, and
light colors lerp along a keyframed ramp and stars fade in near the end.
Chickens get slightly harder to see as it darkens, doubling as a natural
difficulty ramp.

## Camera & input

Camera sits at a fixed point; yaw limited to about ±75°, pitch about
−10°…+30° — a panorama, like the original Moorhuhn's horizontal panning.

- **Desktop:** crosshair follows the mouse; when the pointer enters the outer
  ~15% screen edge the camera pans in that direction. Click shoots a ray
  through the crosshair. `R` reloads. No pointer lock.
- **Touch:** drag rotates the camera; a quick tap (short duration, minimal
  movement) shoots a ray through the tap point. On-screen reload button, as in
  the 2D game.

## Error handling & performance

- WebGL context creation wrapped in try/catch → friendly fallback card with a
  link to `/play/classic`.
- Device pixel ratio capped at 2. Geometry/material disposal on unmount. The
  engine freezes while paused or the tab is hidden (reusing the 2D game's
  blur-pause pattern); the render loop keeps drawing the scene behind the
  pause overlay, and browsers suspend rAF entirely for hidden tabs.
- Chicken meshes cloned from prototypes; particles instanced; no per-frame
  allocations in the hot path.

## Testing

Engine tests need no WebGL — same pattern as the classic game:

- `engine3d`: round lifecycle, reload, band scoring, ray-vs-sphere hit testing
  with deterministic geometry cases, combo raise/reset, golden-chicken spawn
  guarantee, spawn caps.
- `world`: day/night color ramp hits its keyframes and interpolates
  monotonically between them.
- `storage`: keyed API covered; existing key behavior unchanged.
- Hub page render test. The Three.js component itself gets no jsdom mount
  test — WebGL cannot run there, the same trade-off as not pixel-testing the
  2D canvas.

## Out of scope

- Online/global leaderboard (candidate follow-up for both games).
- Pointer-lock FPS controls.
- External 3D model assets (everything is procedural).
