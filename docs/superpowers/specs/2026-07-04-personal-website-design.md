## Design System

- **Colors:** background `#0A0A0A`; surface `#111113`; text `#EDEDED`; muted text `#8F8F94`; accent emerald `#34D399` (hover `#6EE7B7`); 1px borders `#26262A`. WCAG AA contrast minimum for all text.
- **Typography:** Geist Sans (body/headings), Geist Mono (labels, section numbers, code motifs). Loaded via `next/font` (self-hosted, no external requests).
- **Texture:** subtle dot/line grid in hero, thin rules between sections, section numbers in mono (`01 — About`).
- **Spacing:** generous vertical rhythm (~128px between sections desktop, ~80px mobile). Max content width 1100px.

## Motion Spec

- Section entrances: `whileInView` fade + 24px rise, `once: true`, ~0.6s ease-out, children staggered 80ms.
- Hero: orchestrated entrance (name → roles → copy → CTAs), 100ms stagger.
- Rotating role line: typewriter effect — types the title, holds ~2.5s, erases, types the next; mono font, blinking block cursor in accent. Container has a fixed min-height so layout never shifts.
- Micro: buttons spring scale 1.02 + glow on hover; nav links animated underline; timeline dots pulse once when revealed.
- `prefers-reduced-motion`: all movement replaced by simple opacity fades; rotating text becomes static first title.

## Non-Section Requirements

- **SEO:** `metadata` export (title "Daniel Ariel — Senior AI Consultant & Senior Frontend Developer", description from value line); OpenGraph + Twitter card with generated OG image (dark, name + roles); JSON-LD `Person` (name, jobTitle, sameAs LinkedIn, knowsLanguage).
- **Accessibility:** semantic landmarks (`header/main/section/footer`), single `h1`, skip-to-content link, visible focus rings (accent), alt text, AA contrast.
- **Performance:** static output, self-hosted fonts, no external requests, Lighthouse ≥ 95 performance & accessibility.
- **Error handling:** custom 404 page in the same visual language; no other error surface exists (static site, no forms/APIs).
- **CV:** `/cv` print-styled route (light, ATS-friendly layout) rendered once to `public/daniel-ariel-cv.pdf` via headless Chrome during development; nav/contact buttons link the PDF. Daniel reviews PDF before launch.
